PHP 发展史 丹麦爱玩个人主页的青年 1995 27岁

Rasmus Lerdorf（1968 年 11 月 22 日出生）是一名丹麦裔加拿大人程序员。他共同创作并启发了PHP脚本语言，创作了该语言的前两个版本，并参与了由 Jim Winstead（后来创建blo.gs）、Stig Bakken、Shane Caraveo等开发人员领导的后续版本的开发、安迪 Gutmans和Zeev Suraski。他继续为该项目做出贡献。

早年生活和教育

勒多夫出生于格陵兰岛的迪斯科岛，早年移居丹麦。

1980年全家从丹麦移居加拿大，

1983年移居安大略省金城。

1988年毕业于金城中学，

1993年毕业于滑铁卢大学，获学士学位应用科学的系统设计工程。

他为Apache HTTP Server做出了贡献，并将 LIMIT 子句添加到mSQL DBMS。这个 LIMIT 子句的一个变体在大型机关系数据库管理系统中已经存在了十年（比如Oracle Rdb在VAX/VMS上运行，以前来自Digital Equipment Corporation），但显然它尚未被新兴的基于 PC 的数据库所采用。后来它被其他几个与 SQL 兼容的 DBMS 改编。

1995 年发布了 PHP 的第一个版本。

职业生涯

从 2002 年 9 月到 2009 年 11 月，Lerdorf 受雇于Yahoo! Inc.作为基础架构架构工程师。2010 年，他加入WePay以开发他们的应用程序编程接口。整个 2011 年，他都是初创公司的巡回顾问。2012 年 2 月 22 日，他在 Twitter 上宣布他已加入Etsy。2013 年 7 月，Rasmus 加入Jelastic，担任高级顾问，帮助他们开发新技术。

Lerdorf 经常在世界各地的开源会议上发表演讲。在 OSCMS 2007 的主题演讲中，他介绍了当年参加会议的每个项目中的一个安全漏洞。

Lerdorf 还出现在 WeAreDevelopers Conferences 2017 和 2019 上，就 PHP 的历史、2017 年新的 PHP 7 版本以及 PHP 的 25 年发表了演讲。

PHP 开发为了挂自己的个人主页

PHP是一种通用 脚本语言，特别适用于Web 开发。它最初由丹麦-加拿大程序员 Rasmus Lerdorf于 1994 年创建。 PHP参考实现现在由 PHP Group 制作。PHP 最初代表Personal Home Page，但现在代表递归初始化 PHP: Hypertext Preprocessor。

PHP 代码通常由作为模块、守护程序或通用网关接口(CGI) 可执行文件实现的 PHP解释器在Web 服务器上处理。在 Web 服务器上，解释和执行的 PHP 代码的结果（可能是任何类型的数据，例如生成的HTML或二进制图像数据）将构成HTTP响应的全部或一部分。各种网页模板系统、网页内容管理系统和网页框架存在可用于协调或促进该响应的生成。此外，PHP 可用于 Web 上下文之外的许多编程任务，例如独立的图形应用程序和机器人 无人机控制。 PHP 代码也可以直接从命令行执行。

由Zend 引擎提供支持的标准 PHP 解释器是在PHP 许可下发布的免费软件。PHP已经被广泛移植，可以在几乎每一个部署在大多数Web服务器的操作系统和平台，免费的。

PHP 语言在 2014 年之前没有书面正式规范或标准的情况下发展，原始实现作为事实上的标准，其他实现旨在遵循。自 2014 年以来，创建正式 PHP 规范的工作一直在进行。

W3Techs 报告称，截至 2021 年 4 月，“在我们知道服务器端编程语言的所有网站中，有 79.2% 使用 PHP。”

早期历史

Rasmus Lerdorf编写了最初的通用网关接口(CGI) 组件，与Andi Gutmans和Zeev Suraski一起编写了构成 PHP 3的解析器。

PHP 的开发始于 1994 年，当时Rasmus Lerdorf用 C编写了几个通用网关接口(CGI) 程序，他用来维护他的个人主页。他将它们扩展为使用Web 表单并与数据库通信，并将此实现称为“个人主页/表单解释器”或 PHP/FI。

PHP/FI 可用于构建简单、动态的 Web 应用程序。为了加速错误报告和改进代码，Lerdorf 最初于1995 年 6 月 8 日在Usenet讨论组comp.infosystems.www.authoring.cgi上宣布将 PHP/FI 发布为“个人主页工具（PHP 工具）1.0 版” . 这个版本已经具备了 PHP 今天的基本功能。这包括类似 Perl 的变量、表单处理和嵌入 HTML 的能力。该语法类似于说的Perl的，但更简单，更有限，不太一致.

Zend引擎 2个学生的名字组合

在Zend引擎是开源的 脚本引擎是解释的PHP编程语言。它最初是由Andi Gutmans和Zeev Suraski开发的，当时他们还是Technion – Israel Institute of Technology 的学生。他们后来创立了一家名为Zend技术在拉马特甘，以色列。Zend 这个名字是他们的名字 Zeev 和 Andi 的组合。

Zend 引擎的第一个版本出现在 1999 年的 PHP 4 版中。它是用C 语言编写的，作为高度优化的模块化后端，首次可以用于 PHP 之外的应用程序。Zend 引擎为 PHP 语言提供内存和资源管理以及其他标准服务。它的性能、可靠性和可扩展性在 PHP 日益流行的过程中发挥了重要作用。

紧随其后的是作为 PHP 5 核心的Zend Engine 2。

最新版本是Zend Engine 3，最初代号为phpng，它是为 PHP 7 开发的，显着提高了性能。

自 2001 年以来，Zend 引擎的源代码在Zend 引擎许可下免费提供（尽管某些部分在PHP 许可下），[需要引用]作为 php.net 官方发布的一部分，以及官方git存储库或GitHub镜像。各种志愿者为 PHP/Zend 引擎代码库做出贡献。

Zend Engine 在 PHP 内部用作编译器和运行时引擎。PHP 脚本被加载到内存中并编译成 Zend操作码。执行这些操作码并将生成的 HTML 发送到客户端。

要实现 Web 脚本解释器，您需要三个部分：

解释器部分分析输入代码，翻译它并执行它。
功能部分实现了语言的功能（它的功能等）。
接口部分与 Web 服务器等对话。
Zend 完成了第 1 部分和第 2 部分的一些内容；PHP 需要第 2 和第 3 部分。

Zend 本身实际上仅形成了语言核心，通过一些预定义函数在其非常基础的基础上实现 PHP。

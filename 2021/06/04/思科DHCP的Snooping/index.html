思科DHCP的Snooping

发表于 2021-06-04 更新于 2021-06-06 分类于 Network ， Switch 阅读次数：

启用DHCP Snooping的作用
一句话：防止非法的DHCP服务器

哲学思想

  • 防止非法DHCP服务器snooping
  • 防止非法DHCP客户端snooping limit

信任和非信任端口的Action

  • 信任接口：接收DHCP服务器响应的DHCP ACK和DHCP Offer报文。设备只会将DHCP客户端的DHCP请求报文通过信任接口发送给合法的DHCP服务器。
  • 非信任接口：接收到DHCP服务器响应的DHCP ACK和DHCP Offer报文后，直接干掉。

DHCP Snooping要应用在哪里

DHCP Snooping的配置命令

全局启动
Switch(config)# ip dhcp snooping
Enables DHCP snooping globally.

基于VLAN启用
Switch(config)# ip dhcp snooping vlan vlan-range

接口下启用
Switch(config-if)# ip dhcp snooping trust

启用 插入DHCP option-82 信息，DHCP option82会插入在DHCP Request信息里面
Switch(config)# ip dhcp snooping information option

启用 接口收到DHCP包率的限制pps
Switch(config-if)# ip dhcp snooping limit rate 80

查看接口DHCP包率限制
show ip dhcp snooping | begin pps

DHCP OPTION 82

在传统的DHCP动态分配IP地址过程中，DHCP Server不能够根据DHCP请求报文关联到终端的具体物理位置，同一VLAN的用户得到的IP地址所拥有的权限是完全相同的。OPTION 82；包含两个常用子选项Circuit ID和Remote ID。其中Circuit ID子选项主要用来标识客户端所在的VLAN、接口等信息，Remote ID子选项主要用来标识客户端接入的设备，一般为设备的MAC地址。

记录了DHCP Client的位置信息。开启DHCP Snooping的设备通过在DHCP请求报文中添加82选项，将DHCP Client的精确物理位置信息传递给DHCP Server，从而使得DHCP Server能够为主机分配合适的IP地址和其他配置信息，联动安全策略管理器实现对客户端的安全控制。

ssh隧道

发表于 2021-06-06 分类于 Linux 阅读次数：

通过 ssh 的端口转发或者隧道 (tunneling) 功能，可以实现: 加密 SSH Client 端至 SSH Server 端之间的通讯数据.突破防火墙的限制完成一些之前无法建立的 TCP 连接。

SSH隧道tunnel的3个种类

  1. 本地转发
  2. 远程转发
  3. 动态转发

SSH隧道的三种角色

  1. SSH客户端
  2. SSH服务端
  3. 应用

本地转发

应用场景：

  1. 自己电脑 内网直接访问数据库服务器3306,安全原因，被防火墙阻挡，
  2. 自己电脑 访问外网某FTP服务器，但是21被防火墙阻挡, 但是可以SSH公网的自己的服务器

命令：

Local 为ssh客户端一方
remote为ssh服务端一方

ssh -L local_port:remote_address:remote_port username@server.com

ssh -L 8888:localhost:1234 bob@ssh.youroffice.com
ssh -L 8888:192.168.1.111:1234 bob@ssh.youroffice.com

远程转发

应用场景：

命令：

ssh -R remote_port:local_address:local_port username@server.com

ssh -R 8888:localhost:1234 bob@ssh.youroffice.com

动态转发

应用场景：

命令：

ssh -D local_port username@server.com

ssh -D 8888 bob@ssh.yourhome.com

参数介绍

如何关闭这个SSH隧道

方法1：
ps aux | grep 3306 找到pid
kill kill pid

方法2：
终止你从终端打开的所有ssh会话。
sudo killall ssh

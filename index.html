思科DHCP的Snooping

发表于 2021-06-04 分类于 Network ， Switch

启用DHCP Snooping的作用

一句话：防止非法的DHCP服务器

哲学思想

  • 防止非法DHCP服务器snooping
  • 防止非法DHCP客户端snooping limit

信任和非信任端口的Action

  • 信任接口：接收DHCP服务器响应的DHCP ACK和DHCP Offer报文。设备只会将DHCP客户端的DHCP请求报文通过信任接口发送给合法的DHCP服务器。
  • 非信任接口：接收到DHCP服务器响应的DHCP ACK和DHCP Offer报文后，直接干掉。

DHCP Snooping要应用在哪里

DHCP Snooping的配置命令

全局启动
Switch(config)# ip dhcp snooping
Enables DHCP snooping globally.

基于VLAN启用
Switch(config)# ip dhcp snooping vlan vlan-range

接口下启用
Switch(config-if)# ip dhcp snooping trust

启用 插入DHCP option-82 信息，DHCP option82会插入在DHCP Request信息里面
Switch(config)# ip dhcp snooping information option

启用 接口收到DHCP包率的限制pps
Switch(config-if)# ip dhcp snooping limit rate 80

查看接口DHCP包率限制
show ip dhcp snooping | begin pps

DHCP OPTION 82

在传统的DHCP动态分配IP地址过程中，DHCP Server不能够根据DHCP请求报文关联到终端的具体物理位置，同一VLAN的用户得到的IP地址所拥有的权限是完全相同的。OPTION 82；包含两个常用子选项Circuit ID和Remote ID。其中Circuit ID子选项主要用来标识客户端所在的VLAN、接口等信息，Remote ID子选项主要用来标识客户端接入的设备，一般为设备的MAC地址。

记录了DHCP Client的位置信息。开启DHCP Snooping的设备通过在DHCP请求报文中添加82选项，将DHCP Client的精确物理位置信息传递给DHCP Server，从而使得DHCP Server能够为主机分配合适的IP地址和其他配置信息，联动安全策略管理器实现对客户端的安全控制。

clear capwap ap ip address

AP#capwap ap erase all

(Cisco Controller) >config exclusionlist add 00:11:22:33:44:55

从黑名单删除一个用户MAC
(Cisco Controller) >config exclusionlist delete 00:11:22:33:44:55

打掉用户无线连接

什么是npm? 为什么要使用npm?

NPM是随同NodeJS的包管理工具

什么是CNPM？

CNPM全称为”China Node Package Manager”，即中国的包管理镜器。其原理是用一个完整npmjs.org镜像，同步频率目前为10分钟一次以保证尽量与官方服务同步。

国内的NPM软件源有很多，其中阿里云的软件源就是一个不错的选择，本片笔记就是使用阿里的软件源。

npm 的包安装分为本地安装（local）、全局安装（global）两种

npm install说明 常用简写

npm install=npm i。在git clone项目的时候，项目文件中并没有 node_modules文件夹，项目的依赖文件可能很大。直接执行，npm会根据package.json配置文件中的依赖配置下载安装。

-global=-g，全局安装，安装后的包位于系统预设目录下
–save=-S，安装的包将写入package.json里面的dependencies，dependencies：生产环境需要依赖的库
–save-dev=-D，安装的包将写入packege.json里面的devDependencies，devdependencies：只有开发环境下需要依赖的库
